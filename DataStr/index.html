Core Concepts

Data Structure

The way data is stored, organised, and accessed. Programming imitates real-life organisation — a list of students' names is a list data structure.

Algorithm

A set of step-by-step instructions to solve a problem. Like a recipe — each step is clearly defined. In CS, algorithms operate on data structures.

Primitive Data Structures

Basic language-provided types for single values: int, float, char, bool.

Abstract Data Structures

Higher-level structures built from primitives: arrays, linked lists, stacks, queues, trees, graphs.

Recursion

A function that calls itself to break a problem into smaller sub-problems. Two essential parts:

  • Base case — stops the recursion
  • Recursive case — calls itself with a smaller input

DSA is used in:

  • Operating systems, databases, web apps, ML, games
  • GPS route optimisation, search engines, scheduling, cryptography

Recursion Call Stack — factorial(4)

factorial(4)→4 × factorial(3)
factorial(3)→3 × factorial(2)
factorial(2)→2 × factorial(1)
factorial(1)→BASE CASE: returns 1
← unwind:1×2=2, 2×3=6, 6×4 = 24 ✓

Factorial — Recursion
def factorial(n):
    if n == 0 or n == 1:          # base case
        return 1
    return n * factorial(n - 1)    # recursive case

print(factorial(4))  # → 24 (factorial of 4)
Fibonacci — Recursion (0-indexed)
def F(n):
    if n <= 1:               # base cases: F(0)=0, F(1)=1
        return n
    return F(n - 1) + F(n - 2)

print(F(7))  # → 13 (next Fibonacci value)
Fibonacci — For Loop (iterative, more efficient)
prev1, prev2 = 1, 0
print(0)
print(1)

for _ in range(18):
    new_fib = prev1 + prev2
    print(new_fib)
    prev2 = prev1
    prev1 = new_fib

Sum of List
def sum_list(lst):

        return 0
    return lst[0] + sum_list(lst[1:])

print(sum_list([3, 7, 2, 5]))  # → 17
Length of List (no len())
def list_length(lst):
    if lst == []:
        return 0
    return 1 + list_length(lst[1:])

print(list_length([4, 8, 6, 1]))  # → 4
Find Maximum in List
def find_max(lst):
    if len(lst) == 1:          # base case
        return lst[0]
    rest_max = find_max(lst[1:])
    return lst[0] if lst[0] > rest_max else rest_max

print(find_max([3, 9, 1, 7]))  # → 9
Reverse a List
def reverse_list(lst):
    if lst == []:
        return []
    return reverse_list(lst[1:]) + [lst[0]]

print(reverse_list([1, 2, 3, 4]))  # → [4, 3, 2, 1]
Matrix Traversal — Recursion
def recursive_matrix(matrix, i=0, j=0):
    if i == len(matrix):              # all rows done → base case
        return
    if j == len(matrix[i]):           # end of row → next row
        recursive_matrix(matrix, i + 1, 0)
        return
    print(matrix[i][j], end=" ")
    recursive_matrix(matrix, i, j + 1)

m = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
recursive_matrix(m)  # → 1 2 3 4 5 6 7 8 9
⚠ Correction from Sonnet — Matrix Traversal

The lecture's version used return recursive_matrix(...) for the row-advance case. Removing the return before the recursive call is fine, but the original also had no explicit return after the recursive call which could cause duplicate execution in some interpretations. The version above uses return after the recursive call to be safe and clear.

Linked Lists

Lectures 5 & 6 — Pointers, Singly Linked Lists, Doubly Linked Lists, Circular Lists.

What to notice

The lectures explain linked lists as nodes connected by references. Before the code, check the schematic: head is the entry point, tail is the end marker, next moves forward, and in doubly linked lists prev lets you move backward too.

Core Concepts

What is a Linked List?

A sequence of nodes where each node holds data and a reference (pointer) to the next node. Unlike Python lists, elements do not need to be stored in sequential memory.

Advantages
  • No sequential storage space required
  • Can grow arbitrarily as nodes are added
  • Efficient insertions and deletions (no element shifting)
Disadvantages
  • No direct index access (no lst[3])
  • Each node needs extra space for the pointer
  • Search is always O(n) — must traverse from head

Time Complexity

OperationSinglyDoublyNotes
TraversalO(n)O(n)Visit every node
SearchO(n)O(n)Linear scan
AppendO(n) or O(1)O(1)O(1) if tail pointer stored
DeleteO(n)O(n)Must find node first
SizeO(n) or O(1)O(1)O(1) if size attribute stored

Singly Linked List

Linked list diagram
Singly linked list
head[ data | next ]
→ next
node[ data | next ]
→ next
node[ data | None ]
The data lives in each node. The next field points to the following node, and the last node points to None.
Doubly linked list
None[ prev | data | next ]
← prev
node[ prev | data | next ]
→ next
node[ prev | data | None ]
prev points to the previous node, next points to the next node, so you can move in both directions.
Circular linked list
head[ data | next ]
→ next
node[ data | next ]
→ next
last[ data | next ]
↺ back to head
There is no None at the end; the last node links back to the first node.

Linked list schematic
Singly linked list
head → [ data | next ] → [ data | next ] → [ data | None ]

Before you read the code

A node is the basic building block of linked structures. Each node stores the data and the reference to the next node.

Node class
class Node:
    def __init__(self, data=None):
        self.data = data
        self.next = None
SinglyLinkedList — Full Implementation
class SinglyLinkedList:
    def __init__(self):
        self.tail = None
        self.head = None
        self.size = 0

    def append(self, data):
        node = Node(data)
        if self.head:
            self.head.next = node
            self.head = node
        else:
            self.tail = node
            self.head = node
        self.size += 1

    def iter(self):                    # generator for clean traversal
        current = self.tail
        while current:
            val = current.data
            current = current.next
            yield val

    def search(self, data):
        for node in self.iter():
            if data == node:
                return True
        return False

    def delete(self, data):
        current = self.tail
        prev = None
        while current:
            if current.data == data:
                if prev is None:           # deleting the tail (first) node
                    self.tail = current.next
                else:
                    prev.next = current.next
                self.size -= 1
                return
            prev = current
            current = current.next

    def clear(self):
        self.tail = None
        self.head = None

# --- Usage ---
words = SinglyLinkedList()
words.append("eggs")
words.append("ham")
words.append("spam")
for w in words.iter():
    print(w)
⚠ Correction from Sonnet — delete() method

The lecture's delete() initialised prev = self.tail (same as current), which means when deleting the tail node the condition current == self.tail fires correctly, but for the next node the prev.next = current.next also works only by accident. The corrected version above uses prev = None and checks if prev is None — this is the standard and correct pattern that handles all cases clearly.

Doubly Linked List (Lecture 6)

Node class — Doubly Linked
class Node:
    def __init__(self, data=None, next=None, prev=None):
        self.data = data
        self.next = next
        self.prev = prev
DoublyLinkedList — Full Implementation
class DoublyLinkedList:
    def __init__(self):
        self.head = None
        self.tail = None
        self.count = 0

    def append(self, data):
        new_node = Node(data)
        if self.head is None:
            self.head = new_node
            self.tail = new_node
        else:
            new_node.prev = self.tail
            self.tail.next = new_node
            self.tail = new_node
        self.count += 1

    def delete(self, data):
        current = self.head
        while current:
            if current.data == data:
                if current == self.head:
                    self.head = current.next
                    if self.head:
                        self.head.prev = None
                elif current == self.tail:
                    self.tail = current.prev
                    self.tail.next = None
                else:
                    current.prev.next = current.next
                    current.next.prev = current.prev
                self.count -= 1
                return
            current = current.next

    def contain(self, data):
        current = self.head
        while current:
            if current.data == data:
                return True
            current = current.next
        return False

    def display_forward(self):
        current = self.head
        while current:
            print(current.data, end=" → ")
            current = current.next
        print("None")

    def display_backward(self):
        current = self.tail
        while current:
            print(current.data, end=" → ")
            current = current.prev
        print("None")
⚠ Correction from Sonnet — contain() method (Lecture 6, slide 9)

The lecture's contain() used yield inside a while loop and then return True/False after the loop, which is broken: once you use yield, the function becomes a generator and can never reach the return True/False statements. The corrected version above removes the generator pattern and uses a direct comparison loop — this is the correct and working approach.

Circular List (Lecture 6)

CircularList — Append and Iterate
class CircularList:
    def __init__(self):
        self.head = None
        self.tail = None
        self.size = 0

    def append(self, data):
        node = Node(data)
        if self.head:
            self.head.next = node
            self.head = node
        else:
            self.head = node
            self.tail = node
        self.head.next = self.tail     # circular link: last → first
        self.size += 1

    def iter(self):
        current = self.tail
        while current:
            val = current.data
            current = current.next
            yield val
            if current == self.tail:   # stop when we loop back
                break

# --- Usage (safe iteration with counter) ---
words = CircularList()
words.append('eggs')
words.append('ham')
words.append('spam')
counter = 0
for word in words.iter():
    print(word)
    counter += 1
    if counter > 1000:
        break
⚠ Correction from Sonnet — CircularList append() (Lecture 6, slide 12)

The lecture's append() sets self.head.prev = self.tail which implies a doubly-circular structure, but the class was defined as singly-circular (only next). The .prev assignment would cause an AttributeError with a basic Node(data) that has no prev. The corrected version only sets self.head.next = self.tail for the circular link, which is correct for a singly-circular list. If you want doubly-circular, use the doubly Node class with a prev attribute.

Stacks (Lecture 7)

Stack — LIFO

A Last-In, First-Out structure. The last element added is the first one removed. Think of a stack of plates — you can only access the top one.

Stack Operations
MethodDescription
S.push(e)Add element e to top

S.top()Return top element without removing

Before you read the code

The lecture’s stack implementation uses a Python list as the internal storage. The point is to keep the stack interface separate from the list methods.

ArrayStack — Full Implementation (Adapter Pattern)
class Empty(Exception):
    pass


class ArrayStack:
    def __init__(self):
        self._data = []

    def __len__(self):
        return len(self._data)

    def is_empty(self):
        return len(self._data) == 0

    def push(self, e):
        self._data.append(e)

    def top(self):
        if self.is_empty():
            raise Empty("Stack is empty")
        return self._data[-1]

    def pop(self):
        if self.is_empty():
            raise Empty("Stack is empty")
        return self._data.pop()

# --- Usage ---
S = ArrayStack()
S.push(10)
S.push(20)
S.push(30)
print(S.top())    # → 30
S.pop()
print(len(S))    # → 2
Matching Delimiters — Stack Application
def is_matched(expr):
    lefty  = '({['
    righty = ')]}'
    S = ArrayStack()
    for c in expr:
        if c in lefty:
            S.push(c)
        elif c in righty:
            if S.is_empty():
                return False
            if righty.index(c) != lefty.index(S.pop()):
                return False
    return S.is_empty()

print(is_matched("[(5+x)*(y+3)]"))  # → True
print(is_matched("[(1+2)"))          # → False
Reverse a String — Stack Application
def reverse_string(s):
    S = ArrayStack()
    for ch in s:
        S.push(ch)
    result = ""
    while not S.is_empty():
        result += S.pop()
    return result

print(reverse_string("hello"))  # → "olleh"

Queues (Lecture 7)

Queue — FIFO

A First-In, First-Out structure. The first element added is the first one removed. Think of a line of people — the person waiting longest is served first.

Before you read the code

The queue uses circular indexing so enqueue and dequeue do not need to shift all items each time. Watch how _front and _size move together.

ArrayQueue — Circular Implementation
class ArrayQueue:
    DEFAULT_CAPACITY = 10

    def __init__(self):
        self._data  = [None] * ArrayQueue.DEFAULT_CAPACITY
        self._size  = 0
        self._front = 0

    def __len__(self):
        return self._size

    def is_empty(self):
        return self._size == 0

    def first(self):
        if self.is_empty():
            raise Empty("Queue is empty")
        return self._data[self._front]

    def enqueue(self, e):
        avail = (self._front + self._size) % len(self._data)
        self._data[avail] = e
        self._size += 1

    def dequeue(self):
        if self.is_empty():
            raise Empty("Queue is empty")
        answer = self._data[self._front]
        self._data[self._front] = None
        self._front = (self._front + 1) % len(self._data)
        self._size -= 1
        return answer

# --- Usage ---
Q = ArrayQueue()
Q.enqueue("Alice")
Q.enqueue("Bob")
Q.enqueue("Carol")
print(Q.dequeue())  # → Alice (FIFO)
print(Q.first())    # → Bob
⚠ Correction from Sonnet — ArrayQueue capacity

The lecture's circular queue implementation does not handle the case where the queue becomes full (i.e., _size == len(_data)). If you keep enqueuing beyond the initial capacity without a resize step, enqueue() silently overwrites existing data. In a production implementation you would double the internal array when full. For exam purposes, just be aware of this limitation.

Hash Tables (Lecture 8)

What is Hashing?

Converting data of arbitrary size (like a string) into a fixed-size integer (the hash). A hash table uses this integer as an index for O(1) average lookup.

A Python dict is internally implemented using a hash table.

Hash Function Properties
  • Uniform — distributes keys evenly across buckets
  • Fast — computed in O(1)
  • Deterministic — same key always maps to same index
  • Minimises collisions
Collision

When two different keys produce the same hash index. Collisions are unavoidable in practice. Resolution strategies:

  • Chaining — each slot holds a list of elements. Search ≈ O(1 + α) where α = n/m
  • Open Addressing — probe for the next free slot: Linear, Quadratic, Double Hashing

Before you read the code

The lecture builds hash tables by mapping a key to a bucket index. If two keys land in the same bucket, chaining stores both in a small list.

Simple Hash Table with Chaining
class HashTable:
    def __init__(self, m=11):
        self.m    = m
        self.data = [[] for _ in range(m)]  # list of chains

    def _hash(self, key):
        return key % self.m                    # division method

    def insert(self, key, value):
        idx = self._hash(key)
        for i, (k, v) in enumerate(self.data[idx]):
            if k == key:
                self.data[idx][i] = (key, value)  # update
                return
        self.data[idx].append((key, value))       # new entry

    def search(self, key):
        idx = self._hash(key)
        for k, v in self.data[idx]:
            if k == key:
                return v
        return None

# --- Usage ---
ht = HashTable()
ht.insert(10, "Alice")
ht.insert(21, "Bob")    # 21 % 11 = 10 → collision → chaining
print(ht.search(10))  # → Alice
print(ht.search(21))  # → Bob
⚠ Correction from Sonnet — Load Factor α definition (Lecture 8, slide 23)

The lecture states the load factor example as α = 30/10 = 3 and says "On average, each chain contains 3 elements." This is technically correct for chaining, but the lecture also says "Search ≈ O(1)" when α is constant. This only holds when α is bounded by a small constant (e.g., α ≤ 1). When α = 3 the average chain length is 3 so search is O(3) = O(1) only in a constant sense — but for large α, performance degrades significantly. For Open Addressing, α must be kept strictly below 1 (table cannot be full).
