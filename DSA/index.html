Question 1

A sorting method with Big-Oh complexity \(O(n\log n)\) spends exactly \(1\) millisecond to sort \(1000\) data items. Assuming that the running time \(T(n)\) for sorting \(n\) items is directly proportional to \(n\log n\), i.e., \(T(n)=cn\log n\), derive a formula for \(T(n)\) in terms of the known time \(T(n_0)\) for sorting \(n_0\) items, and estimate how long this method will take to sort \(1{,}000{,}000\) items.

Since \(T(n)=cn\log n\), and for a known input size \(n_0\), \[ T(n_0)=cn_0\log n_0, \] we obtain \[ c=\frac{T(n_0)}{n_0\log n_0}. \] Substituting into the expression for \(T(n)\), \[ T(n)=\frac{T(n_0)}{n_0\log n_0}\,n\log n. \] Therefore, \[ \boxed{T(n)=T(n_0)\frac{n\log n}{n_0\log n_0}}. \] Given \(n_0=1000\) and \(T(1000)=1\text{ ms}\), \[ T(1{,}000{,}000) = 1\text{ ms}\cdot \frac{1{,}000{,}000\log(1{,}000{,}000)} {1000\log(1000)}. \] Using base-10 logarithms, \[ \log(1000)=3,\qquad \log(1{,}000{,}000)=6, \] hence \[ T(1{,}000{,}000) = 1\text{ ms}\cdot \frac{1{,}000{,}000\cdot 6} {1000\cdot 3} = 2000\text{ ms} = 2\text{ s}. \] Thus, the estimated running time for sorting \(1{,}000{,}000\) items is \[ \boxed{2\text{ seconds}}. \]

Question 2

A quadratic algorithm with processing time \(T(n)=cn^2\) spends \(T(n_0)\) seconds for processing \(n_0\) data items. How much time will be spent for processing \(n=5000\) data items, assuming that \(n_0=100\) and \(T(100)=1\text{ ms}\)?

Since the algorithm is quadratic, \[ T(n)=cn^2. \] Using the known value \(T(n_0)\), \[ T(n_0)=cn_0^2, \] so \[ c=\frac{T(n_0)}{n_0^2}. \] Substituting into \(T(n)\), \[ T(n)=T(n_0)\frac{n^2}{n_0^2}. \] With \(n=5000\), \(n_0=100\), and \(T(100)=1\text{ ms}\), \[ T(5000) = 1\text{ ms}\cdot \frac{5000^2}{100^2} = 1\text{ ms}\cdot \left(\frac{5000}{100}\right)^2 = 1\text{ ms}\cdot 50^2 = 2500\text{ ms}. \] Since \[ 2500\text{ ms}=2.5\text{ s}, \] the processing time is \[ \boxed{T(5000)=2500\text{ ms}=2.5\text{ seconds}.} \]

Question 3

An algorithm with time complexity \(O(f(n))\) and processing time \(T(n)=cf(n)\), where \(f(n)\) is a known function of \(n\), spends \(10\) seconds to process \(1000\) data items. How much time will be spent to process \(100{,}000\) data items if (a) \(f(n)=n\) and (b) \(f(n)=n^2\)?

Since \(T(n)=cf(n)\), and \(T(n_0)=cf(n_0)\), we have \[ T(n)=T(n_0)\frac{f(n)}{f(n_0)}. \] Given \[ n_0=1000,\qquad T(1000)=10\text{ s}, \] we evaluate the two cases. \textbf{(a) Linear complexity: \(f(n)=n\)} \[ T(100{,}000) = 10\text{ s}\cdot \frac{100{,}000}{1000} = 10\cdot 100 = 1000\text{ s}. \] Therefore, \[ \boxed{T(100{,}000)=1000\text{ s}\approx 16.67\text{ min}.} \] \textbf{(b) Quadratic complexity: \(f(n)=n^2\)} \[ T(100{,}000) = 10\text{ s}\cdot \frac{(100{,}000)^2}{(1000)^2} = 10\cdot 100^2 = 10\cdot 10{,}000 = 100{,}000\text{ s}. \] Therefore, \[ \boxed{T(100{,}000)=100{,}000\text{ s}\approx 27.78\text{ h}\approx 1.16\text{ days}.} \]

Question 8

Prove that \(T(n)=a_0+a_1n+a_2n^2+a_3n^3\) is \(O(n^3)\) using the formal definition of Big-Oh notation.

For \(n \ge 1\), we have \(1 \le n \le n^2 \le n^3\). Therefore,

\[ T(n)=a_0+a_1n+a_2n^2+a_3n^3 \le (|a_0|+|a_1|+|a_2|+|a_3|)n^3. \]

Choose

\[ c = |a_0|+|a_1|+|a_2|+|a_3|, \qquad n_0 = 1. \]

Then for all \(n \ge n_0\), \(T(n) \le cn^3\). Hence, by definition, \(T(n)=O(n^3)\).

Question 9

Algorithms A and B spend exactly \(T_A(n)=0.1n^2\log_{10}n\) and \(T_B(n)=2.5n^2\) microseconds, respectively. Which is better in the Big-Oh sense, when does it actually become faster, and what do you recommend for \(n \le 10^9\)?

Question 1

Write an algorithm that takes as input a list of \(n\) integers in nondecreasing order and produces the list of all values that occur more than once. Estimate the worst-case complexity of your algorithm.

Scan the sorted list once and compare each element with the next one. Whenever two consecutive elements are equal, report the value once and skip repeated copies of it.

Duplicates(A)
1  D = empty list
2  i = 0
3  while i < n - 1:
4    if A[i] == A[i+1]:
5      append A[i] to D
6      while i < n - 1 and A[i] == A[i+1]:
7        i = i + 1
8    i = i + 1
9  return D

The list is already sorted, so every comparison is constant time and each element is examined at most once. The worst-case complexity is \(\Theta(n)\).

Let \(u\) and \(v\) be adjacent vertices in a graph \(G\) of order \(n\). Prove that the edge \(uv\) belongs to at least \(\deg(u)+\deg(v)-n\) triangles in \(G\).

\[ n \ge |N(u) \cup N(v)| = |N(u)| + |N(v)| - |N(u) \cap N(v)|. \]

Therefore,

\[ |N(u) \cap N(v)| \ge \deg(u) + \deg(v) - n. \]

Hence, the edge \(uv\) belongs to at least \(\deg(u)+\deg(v)-n\) triangles.

Question 1

Several families go out to dinner together. They would like to sit at tables so that no two members of the same family are at the same table. Model this as a max-flow problem. Build the following flow network:

  • A sink \(t\).

A valid seating assignment exists if and only if the maximum flow in this network equals \(\sum_i p_i\) (the total number of people). Max-flow runs in polynomial time, so the algorithm is polynomial.

Question 9

Let \(G\) be a network with source \(s\), sink \(t\), and integer capacities. Prove that an edge \(e\) is saturated in every maximum \(s\)-\(t\) flow if and only if decreasing the capacity of \(e\) by 1 would decrease the maximum value of an \(s\)-\(t\) flow in \(G\).

Forward direction. If every maximum flow saturates \(e\), then reducing \(e\)'s capacity by 1 lowers the capacity of every maximum flow by at least 1, so the maximum flow value decreases.

Reverse direction. If decreasing \(c(e)\) by 1 decreases the maximum flow value, then every maximum flow in the original network must have used all of \(e\)'s capacity. Otherwise, if some maximum flow left at least one unit unused on \(e\), that same flow would still be feasible after the decrease, contradicting the drop in the maximum value.

Question 10

You are given a unit-capacity network and a parameter \(k\). Delete exactly \(k\) edges so that the maximum \(s\)-\(t\) flow decreases as much as possible. Give a polynomial-time algorithm.

Since all capacities are 1, the maximum flow equals the size of a minimum cut. To reduce the maximum flow as much as possible by deleting \(k\) edges, choose edges that hit as many \(s\)-\(t\) paths as possible. A polynomial-time approach is:

  1. Compute a maximum flow / minimum cut.
  2. Use the residual network to identify critical edges on all minimum cuts.
  3. Choose up to \(k\) edges whose deletion reduces the minimum cut value as much as possible, using repeated min-cut computations or a matroid-style greedy method on the unit-capacity structure.

Because capacities are unit and each edge deletion changes the max flow by at most 1, the problem can be solved with a polynomial number of max-flow computations.

Question 11

In a flow network with vertex capacities, give a polynomial-time algorithm to find a maximum \(s\)-\(t\) flow, define an \(s\)-\(t\) cut for vertex-capacitated networks, and show that the Max-Flow Min-Cut Theorem still holds.

Convert the vertex-capacitated network into an edge-capacitated one by splitting each vertex \(v\neq s,t\) into \(v_{in}\) and \(v_{out}\), and add an edge \(v_{in}\to v_{out}\) with capacity \(c(v)\). Every original edge \((u,v)\) becomes an edge \(u_{out}\to v_{in}\) with the same capacity.

Now run any polynomial-time max-flow algorithm on the transformed network. The resulting flow respects the vertex capacities because all flow through \(v\) must pass through \(v_{in}\to v_{out}\).

A vertex cut is a set of vertices whose removal disconnects \(s\) from \(t\), with cut capacity equal to the sum of capacities of the removed vertices. The transformed edge network has a cut of the same value, so the standard Max-Flow Min-Cut Theorem applies directly.

Core Proof Methods (The "Big Five")

— Direct Proof

  1. Assume \(p\) is true.
  2. Use definitions, axioms, earlier theorems.
  3. Deduce \(q\) step by step.
  4. Conclude "Therefore \(p \to q\)".

Example: If \(n = 2k\), then \(n^2 = 4k^2 = 2(2k^2)\) is even.

— Proof by Contraposition

  1. Prove \(\lnot q \to \lnot p\).
  2. Assume \(\lnot q\), derive \(\lnot p\).
  3. Conclude \(p \to q\) holds.

Example: If \(n = 2k+1\) is odd, then \(n^2 = 2(2k^2+2k)+1\) is odd.

— Proof by Contradiction

  1. Assume \(\lnot p\) or \(p \land \lnot q\).
  2. Derive contradiction \(r \land \lnot r\).
  3. Conclude original statement is true.

Example: \(\sqrt{2}\) irrational — assuming \(\sqrt{2}=a/b\) in lowest terms forces both \(a,b\) even. Contradiction. □

— Proof by Cases

  1. Identify mutually exclusive, exhaustive cases.
  2. Prove \(c_i \to q\) for each.

— Mathematical Induction

Weak: Base \(P(n_0)\) → IH \(P(k)\) → Step \(P(k+1)\).   Strong: Assume \(P(a),\dots,P(k)\); prove \(P(k+1)\).

Examples: \(\sum_{i=1}^n i = \frac{n(n+1)}{2}\)  ·  \(3 \mid (n^3-n)\)  ·  \(2^n > n^2\) for \(n \ge 5\).

§5 — Big-O   (Upper Bound)

Formal Definition
\[ T(n) = O(f(n)) \iff \exists\,c>0,\,n_0\ \text{ s.t. }\ \forall n \ge n_0,\quad 0 \le T(n) \le c\cdot f(n) \]

§11 — Divide-and-Conquer Paradigm

Definition

Given an input of size \(n\), imagine a magic box that solves the same problem for any input strictly smaller than \(n\). Your job: break the big input into smaller pieces, run them through the box, then stitch the results together. The box is not magic — it is the same algorithm called recursively until a trivial base case is reached.

Every divide-and-conquer algorithm performs three steps in the recursive case:

  1. Divide — split the problem into smaller instances of the same problem.
  2. Conquer — solve each subproblem recursively.
  3. Combine — merge the subproblem solutions into a solution for the original.

Recursion bottoms out at the base case, where the input is small enough to solve directly (e.g. \(n\le1\)).

§12 — MergeSort

Algorithm Idea

Split the array in half, recursively sort both halves, then merge the two sorted halves into one sorted array.

MergeSort Pseudocode

MergeSort(A)
1   n = len(A)
2   if n <= 1:               # base case: already sorted
3       return A
4   L = MergeSort(A[0:n/2])   # sort left half
5   R = MergeSort(A[n/2:n])   # sort right half
6   return Merge(L, R)        # merge the two halves

Python notation: \(A[:n/2] = [A[0],\ldots,A[n/2-1]]\), \(A[n/2:] = [A[n/2],\ldots,A[n-1]]\). Integer division: \(n/2\) means \(\lfloor n/2 \rfloor\).

Merge Pseudocode

Merge(L, R)   # Assume L and R are already sorted
1   S = List of size n
2   i = 0; j = 0
3   for k in [0, ..., n-1]:
4       if L[i] < R[j]:
5           S[k] = L[i]
6           i = i + 1
7       else:
8           S[k] = R[j]
9           j = j + 1
10   return S

# Note: pseudocode above is incomplete — handle end-of-L or end-of-R!

How Merge works: Keep two pointers, one per sorted half. At each step, copy the smaller of the two pointed-to elements into the result, then advance that pointer. Repeat until all elements are copied.

§13 — Correctness of MergeSort

Correctness of Merge (induction on iterations)

Loop Invariant (IH)

After iteration \(t\) of the for loop, \(S[:t-1]\) contains the \(t\) smallest elements of \(L\) and \(R\) in sorted order. Moreover, \(L[i]\) and \(R[j]\) are the smallest elements of their arrays not yet copied to \(S\).

  • Base case (\(t=0\)): \(i=j=0\), \(S\) empty. \(L[0]\) and \(R[0]\) are smallest in their arrays. ✓
  • Inductive step: By IH, after iteration \(t-1\), \(S[:t-2]\) holds the \(t-1\) smallest elements. WLOG \(L[i]\le R[j]\) in iteration \(t\). Then \(L[i]\) is the smallest not yet copied; appending it gives \(S[:t-1]\) with the \(t\) smallest elements in sorted order. ✓
  • Conclusion: When Merge terminates (\(t=n,\,k=n-1\)), \(S[:n-1]\) contains all elements of \(L\) and \(R\) in sorted order. □

Correctness of MergeSort (strong induction on \(n\))

Inductive Hypothesis

Whenever MergeSort returns an array of size \(\le i\), that array is sorted.

  • Base case (\(i=1\)): Arrays of length 0 or 1 are trivially sorted. ✓
  • Inductive step: For input of length \(i>1\), both halves have length \(<i\). By strong IH, lines 4 and 5 correctly produce sorted \(L\) and \(R\). Since Merge is correct (proved above), MergeSort returns the sorted array. ✓
  • Conclusion: By induction, MergeSort correctly sorts any array of length \(n\). □

§14 — Running Time of MergeSort

Work per node

In a single call to MergeSort with input of size \(k = n/2^t\):

  • Checking base case: \(O(1)\)
  • Computing midpoint (divide step): \(O(1)\)
  • Running Merge: \(O(k)\) — the for loop runs exactly \(k\) times, each iteration \(O(1)\)

Total work per node with input size \(k\): \(O(1)+O(1)+O(k) = O(k)\).

Recursion Tree

Level 0         [          n          ]      1 problem of size n,       work = C·n
Level 1         [  n/2  ] [  n/2  ]               2 problems of size n/2,   work = C·n
Level 2         [n/4][n/4][n/4][n/4]      4 problems of size n/4,   work = C·n
                                  ⋮
Level t         2ᵗ problems of size n/2ᵗ,                        work = 2ᵗ · C·(n/2ᵗ) = C·n
                                  ⋮
Level log n     n problems of size 1,                             work = C·n

Key insight: Work at every level \(t\) equals \(2^t \cdot C\cdot(n/2^t) = C\cdot n\) — constant per level, independent of \(t\). The two exponentials cancel exactly.

Number of levels: \(\log n + 1\) (tree depth is \(\log n\) since size halves each level).

Claim 1 — MergeSort Runtime (Recursion Tree Proof)

Theorem
\[ T_{\text{MergeSort}}(n) = O(n \log n) \]

Proof:

\[ \text{Total} = (\text{ops per level}) \times (\text{number of levels}) \le C\cdot n \cdot (\log n + 1) = C\cdot(n\log n + n) = O(n\log n) \quad \square \]

Recurrence Relation

Recurrence
\[ T(n) \le 2\,T\!\left(\frac{n}{2}\right) + C\cdot n \qquad T(1) \le C \]

where \(C\) is a constant. For \(n=2^k\): \(T(2^k) \le 2\,T(2^{k-1}) + C\cdot 2^k\).

"Plug-and-Pattern-Match" Proof

Unrolling the recurrence \(k\) times (let \(n=2^k\)):

\[ T(2^k) \le 2^i\,T(2^{k-i}) + i\cdot C\cdot 2^k \]

Setting \(i=k\) and using \(T(2^0)=T(1)\le C\):

\[ T(2^k) \le 2^k\cdot C + k\cdot C\cdot 2^k = C(k+1)2^k = O(2^k\cdot k) \]

Since \(n=2^k\) and \(k=\log n\):

\[ T(n) = O(n\log n) \quad \square \]

"Induction-from-the-Ansatz" Proof

Guess \(T(2^k)\le C(k+1)2^k\) and prove by induction.

  • Base (\(k=0\)): \(T(1)\le C = C\cdot1\cdot1 = C(0+1)2^0\). ✓
  • IH: \(T(2^{k-1})\le C\cdot k\cdot 2^{k-1}\).
  • Step: \[ T(2^k)\le 2T(2^{k-1})+C\cdot 2^k \le 2\cdot Ck\cdot 2^{k-1}+C\cdot 2^k = Ck\cdot 2^k + C\cdot 2^k = C(k+1)2^k \quad\square \]

§16 — Practical Notes on MergeSort

  • External sorting: Merge only needs the "tips" (front elements) of each half — great when data doesn't fit in memory.
  • Predictable runtime: \(\Theta(n\log n)\) in all cases — no worst-case blowup unlike InsertionSort.
  • Parallelism: Recursive calls can run in parallel. Naive parallelisation gives \(\Theta(n)\) (sequential merge bottleneck). Parallel merge reduces this to \(\Theta(n/\log^2 n)\) with infinite CPUs.
  • Design question to ask: If I could solve this problem for any portion of the input independently, can I stitch those solutions back together?

§L4-1 — What Is a Graph?

Definition

A graph \(G=(V,E)\) is a set of vertices (nodes) \(V\) and a set of edges \(E\), where each edge connects a pair of vertices. It models pairwise relationships between items.

  • Order of \(G\): the number of vertices \(|V|\).
  • Size of \(G\): the number of edges \(|E|\).
  • A graph of order 0 or 1 is called trivial.
  • Drawing: vertices as circles, edges as lines — only connectivity matters, not geometry.

Origin: Leonhard Euler (1736) solved the Königsberg Seven Bridges Problem by abstracting land masses as vertices and bridges as edges — the first theorem in graph theory.

Trees

A tree is a connected, acyclic undirected graph. Trees are the simplest connected structures and appear everywhere in algorithms.

§T1 — Tree Definition & Properties

Definition

A tree is a graph \(T=(V,E)\) that is connected and acyclic (no cycles).

The following are all equivalent for a graph \(G\) with \(n\) vertices:

  • \(G\) is a tree.
  • \(G\) is connected and has exactly \(n-1\) edges.
  • \(G\) is acyclic and has exactly \(n-1\) edges.
  • Any two vertices are connected by exactly one simple path.
  • \(G\) is connected, but removing any edge disconnects it.
  • \(G\) is acyclic, but adding any edge creates exactly one cycle.
Forest

An acyclic graph (not necessarily connected). Each connected component of a forest is a tree. A forest with \(n\) vertices and \(k\) components has exactly \(n-k\) edges.

Eulerian Paths & Circuits

§E1 — Definitions

§E2 — Euler’s Theorem

Theorem (Euler, 1736)

A connected undirected graph \(G\) has an Eulerian circuit if and only if every vertex has even degree.

A connected undirected graph \(G\) has an Eulerian path (not a circuit) if and only if it has exactly two vertices of odd degree (the path starts at one and ends at the other).

Königsberg answer: All four land masses had odd degree — no Eulerian path exists.

Directed version

  • Eulerian circuit in a digraph: \(G\) is connected (weakly) and \(\deg^+(v)=\deg^-(v)\) for every vertex \(v\).
  • Eulerian path in a digraph: exactly one vertex has \(\deg^+(v)-\deg^-(v)=1\) (start), one has \(\deg^-(v)-\deg^+(v)=1\) (end), all others balanced.

Finding an Eulerian circuit: Hierholzer’s algorithm runs in \(O(m)\) — follow edges greedily, splicing in detours when stuck, until all edges are used.

Note: Eulerian path ↔ every edge once. Hamiltonian path ↔ every vertex once. These are very different problems: Euler is polynomial; Hamiltonian is NP-complete.

Planar Graphs

§P1 — Definition & Euler’s Formula

Definition

A graph is planar if it can be drawn in the plane with no edges crossing. Such a drawing is called a planar embedding.

A planar embedding divides the plane into regions called faces (including the one unbounded outer face).

Theorem — Euler’s Formula for Planar Graphs

For any connected planar graph with \(V\) vertices, \(E\) edges, and \(F\) faces:

\[ V - E + F = 2 \]

Consequence (edge bound): For a simple connected planar graph with \(n\ge 3\) vertices: \(E \le 3V-6\). For triangle-free planar graphs: \(E \le 2V-4\).

Consequence (degree): Every simple planar graph has a vertex of degree \(\le 5\).

§P2 — Kuratowski’s Theorem

Theorem (Kuratowski, 1930)

A graph is planar if and only if it contains no subdivision of \(K_5\) or \(K_{3,3}\) as a subgraph.

  • \(K_5\): complete graph on 5 vertices.
  • \(K_{3,3}\): complete bipartite graph on \(3+3\) vertices.
  • A subdivision of a graph \(H\) replaces edges with paths (inserts new degree-2 vertices).
  • Wagner’s theorem (equivalent): a graph is planar iff it has no \(K_5\) or \(K_{3,3}\) as a minor.

Topological Ordering

§TO1 — Definition & Conditions

Definition

A topological ordering (topological sort) of a directed graph \(G=(V,E)\) is a linear ordering \(v_1, v_2, \ldots, v_n\) of all vertices such that for every directed edge \((u,v)\in E\), \(u\) appears before \(v\) in the ordering.

Theorem

A topological ordering of \(G\) exists if and only if \(G\) is a DAG (Directed Acyclic Graph). A DAG has no directed cycles.

  • A topological ordering may not be unique — multiple valid orderings can exist.
  • Applications: scheduling tasks with dependencies, build systems, course prerequisites, spreadsheet evaluation.
  • Any DAG has at least one vertex with in-degree 0 (a source) — this is always placed first.

BFS & DFS

§L4-9 — BFS (Breadth-First Search)

Definition

BFS explores a graph level by level from source \(s\) using a queue. It computes the shortest path (fewest edges) from \(s\) to every reachable vertex.

BFS(G, s)
1   for each u ≠ s: color[u] = WHITE, d[u] = ∞, π[u] = NIL
2   color[s] = GRAY, d[s] = 0, π[s] = NIL
3   Q = empty queue; Enqueue(Q, s)
4   while Q ≠ ∅:
5       u = Dequeue(Q)
6       for each v ∈ Adj[u]:
7           if color[v] == WHITE:
8               color[v] = GRAY; d[v] = d[u]+1; π[v] = u
9               Enqueue(Q, v)
10      color[u] = BLACK

  • Works on directed and undirected graphs.
\[ T_{\text{BFS}} = O(n + m) \]

Definition

DFS explores as far as possible along each branch before backtracking, using recursion (implicit stack). Records discovery time \(d[v]\) and finish time \(f[v]\).

DFS(G)
1   for each u: color[u] = WHITE, π[u] = NIL
2   time = 0

1   color[u] = GRAY; time++; d[u] = time
2   for each v ∈ Adj[u]:
3       if color[v] == WHITE: π[v] = u; DFS-Visit(G, v)
4   color[u] = BLACK; time++; f[u] = time

Max Flow & Min Cut — Graph Definitions

Graph-theoretic definitions from Lecture 5: flow networks, cuts, residual networks, and the Max-Flow Min-Cut theorem.

§G1 — Flow Network

Definition

A flow network is a directed graph \(G=(V,E)\) with a capacity function \(c:E\to\mathbb{R}_{\ge0}\), a single source \(s\in V\) (no incoming edges), and a single sink \(t\in V\) (no outgoing edges). All other vertices are internal.

  • No edge enters \(s\); no edge leaves \(t\).
  • Every vertex has at least one incident edge.
  • All capacities are integers (in the standard formulation).

§G2 — Flow & Its Value

s-t Flow

A function \(f:E\to\mathbb{R}_{\ge0}\) satisfying:

Capacity Constraint

\[ \forall\,(u,v)\in E \qquad 0 \le f(u,v) \le c(u,v) \]

Flow Conservation

\[ \forall\,v\in V{-}\{s,t\} \quad \sum_{x\in N_{\mathrm{in}}(v)}f(x,v)=\sum_{y\in N_{\mathrm{out}}(v)}f(v,y) \]

Value of a Flow

\[ v(f)=\sum_{x\in N_{\mathrm{out}}(s)}f(s,x)-\sum_{y\in N_{\mathrm{in}}(s)}f(y,s) \]

Max Flow Problem: Find the \(s\)-\(t\) flow \(f\) that maximises \(v(f)\).

§G3 — s-t Cut

Definition

An \(s\)-\(t\) cut is a partition \(V=S\cup T\), \(S\cap T=\emptyset\), \(s\in S\), \(t\in T\). Its capacity counts only edges from \(S\) to \(T\):

\[ c(S,T)=\sum_{\substack{(u,v)\in E\\u\in S,\,v\in T}}c(u,v) \]

Min Cut Problem: Find the \(s\)-\(t\) cut of minimum capacity (cheapest disconnection of \(s\) from \(t\)).

Multiple sources/sinks: Add a super-source with \(\infty\)-capacity edges to all real sources; similarly for a super-sink.

§G4 — Max-Flow Min-Cut Theorem

Lemma 1 — Weak Duality

For any flow \(f\) and any cut \((S,T)\):

\[ v(f)\le c(S,T) \]
Corollary 1

If \(v(f)=c(S,T)\) for some \(f\) and \((S,T)\), then \(f\) is a maximum flow and \((S,T)\) is a minimum cut.

Theorem 1 — Max-Flow Min-Cut
\[ \max_f\,v(f)=\min_{(S,T)}c(S,T) \]

In every flow network, max \(s\)-\(t\) flow value = min \(s\)-\(t\) cut capacity.

§G5 — Residual Network

Definition: \(G_f\)

Given \(G\) and flow \(f\), the residual network \(G_f\) has the same vertices. For each edge \((u,v)\in E\):

ConditionEdge in \(G_f\)TypeResidual capacity
\(f(u,v)<c(u,v)\)\((u,v)\)Forward\(c_f(u,v)=c(u,v)-f(u,v)\)
\(f(u,v)>0\)\((v,u)\)Backward\(c_f(v,u)=f(u,v)\)

\(G_f\) contains only edges with strictly positive residual capacity. Each original edge yields at most 2 edges in \(G_f\), so \(|E_{G_f}|\le 2|E|\).

§G6 — Augmenting Path

Definition

An augmenting path is any path from \(s\) to \(t\) in \(G_f\). Its bottleneck is \(x=\min_{e\in P}c_f(e)\).

\[ f'(u,v)=\begin{cases}f(u,v)+x&(u,v)\in P\text{ (forward)}\\f(u,v)-x&(v,u)\in P\text{ (backward)}\\f(u,v)&\text{otherwise}\end{cases} \] \[ v(f')=v(f)+x>v(f) \]
Lemma 2

If \(t\) is not reachable from \(s\) in \(G_f\), then \(f\) is a max flow. (Let \(S=\) vertices reachable from \(s\); then \(v(f)=c(S,T)\).)

Lemma 3

If \(G_f\) has any path from \(s\) to \(t\), we can augment \(f\) to \(f'\) with \(v(f')>v(f)\).

Graph Representations

Data structures for storing graphs and their trade-offs, from Lecture 4.

§R1 — Adjacency Matrix

Definition

An \(n\times n\) matrix \(A\) where:

\[ A[i][j] = \begin{cases}1 & \text{if }(i,j)\in E \\ 0 & \text{otherwise}\end{cases} \]

For a multigraph: \(A[i][j]\) = number of edges between \(i\) and \(j\).
For a digraph: \(A[i][j]=1\) means arc from \(i\) (source/row) to \(j\) (destination/column).

  • Symmetric for undirected graphs: \(A[i][j]=A[j][i]\).
  • Space: \(\Theta(n^2)\).
  • Edge query \((u,v)\in E\): \(O(1)\).
  • Neighbour query (list all neighbours of \(v\)): \(O(n)\).
  • Slow to add/remove vertices — matrix must be resized/copied.

§R2 — Adjacency List

Definition

An array of \(n\) lists. \(\text{Adj}[v]\) contains all vertices adjacent to \(v\).

For a digraph: \(\text{Adj}[v]\) stores only the out-neighbours of \(v\).

  • Space: \(\Theta(n+m)\) where \(m=|E|\) — efficient for sparse graphs.
  • Neighbour query: \(O(\deg(v))\).
  • Edge query \((u,v)\in E\): \(O(\deg(u))\) — must scan the list.
  • Slow to remove a vertex — must find and delete it from all neighbour lists.
  • Preferred for sparse graphs and most algorithm implementations.

§BFS — Breadth-First Search

BFS(G, s)   # uses queue; finds shortest paths (unweighted)

\[ T_{\text{BFS}} = O(n + m) \]

DFS(G)   # visits all vertices; records timestamps

3   color[u]=BLACK; f[u]=++time

Topological Sort

§TS1 — DFS-Based Topological Sort

Run DFS on the DAG. When a vertex finishes (goes BLACK), prepend it to the output list. The result is a valid topological order.

TopoSort-DFS(G)
1   Run DFS(G) recording finish times f[v]
2   Output vertices in decreasing order of f[v]

Correctness: For every edge \((u,v)\), DFS finishes \(u\) after \(v\) iff \(v\) is not an ancestor of \(u\) — guaranteed in a DAG (no back edges). So \(f[u] > f[v]\) and \(u\) comes first. □

\[ T = O(n + m) \]

§TS2 — Kahn’s Algorithm (BFS-Based)

Kahn(G)
1   Compute in-degree indeg[v] for all v
2   Q = queue of all vertices with indeg[v] == 0
3   order = []
4   while Q ≠ ∅:
5       u = Dequeue(Q); order.append(u)
6       for each v ∈ Adj[u]:
7           indeg[v] -= 1
8           if indeg[v] == 0: Enqueue(Q, v)
9   if len(order) ≠ n: report # cycle detected — not a DAG
10  return order

Cycle detection: If the output has fewer than \(n\) vertices, the graph has a cycle. \(T = O(n+m)\).

Minimum Spanning Tree (MST)

§MST1 — Definition & Properties

Definition

Given a connected, weighted, undirected graph \(G=(V,E,w)\), a spanning tree \(T\) is a subgraph that is a tree covering all \(n\) vertices (so \(n-1\) edges). A minimum spanning tree (MST) is a spanning tree of minimum total weight:

\[ w(T) = \sum_{e\in T} w(e) \quad \text{minimised} \]

If all edge weights are distinct, the MST is unique.

Cut Property

Cut Property

For any cut \((S, V\setminus S)\) of \(G\), the minimum-weight edge crossing the cut is in every MST. (If weights are distinct, it’s in the unique MST.)

Cycle Property

Cycle Property

For any cycle \(C\) in \(G\), the maximum-weight edge of \(C\) is not in any MST. (If weights are distinct.)

§MST2 — Kruskal’s Algorithm

Strategy: Greedy — sort all edges by weight, add each edge if it doesn’t create a cycle. Uses Union-Find to check connectivity.

Kruskal(G, w)
1   Sort edges by weight: \(e_1 \le e_2 \le \cdots \le e_m\)
2   T = ∅   # MST edge set
3   MakeSet(v) for each v ∈ V   # init Union-Find
4   for each edge (u, v) in sorted order:
5       if Find(u) ≠ Find(v):   # u, v in different components
6           T = T ∪ {(u,v)}
7           Union(u, v)
8   return T

Correctness: Each added edge is the lightest crossing some cut — safe by the Cut Property.

\[ T_{\text{Kruskal}} = O(m \log m) = O(m \log n) \quad\text{(dominated by sorting)} \]

§MST3 — Prim’s Algorithm

Strategy: Greedy — grow the MST one vertex at a time, always adding the cheapest edge connecting the current tree to a new vertex. Uses a priority queue.

Prim(G, w, r)   # r = start vertex
1   for each u ∈ V: key[u] = ∞, π[u] = NIL
2   key[r] = 0
3   Q = priority queue of all V (key = key[v])
4   while Q ≠ ∅:
5       u = Extract-Min(Q)   # cheapest edge into tree
6       for each v ∈ Adj[u]:
7           if v ∈ Q and w(u,v) < key[v]:
8               π[v] = u; key[v] = w(u,v)
9               Decrease-Key(Q, v, key[v])
10  return {(π[v], v) : v ≠ r}

key[v] = minimum weight of any edge connecting \(v\) to the current tree. \(\pi[v]\) = the corresponding tree-side endpoint.

Union-Find (Disjoint Set Union)

§UF1 — Definition & Operations

Definition

A Union-Find (DSU) data structure maintains a partition of a set into disjoint subsets and supports two operations efficiently:

  • Find(x): return the representative (root) of the set containing \(x\).
  • Union(x, y): merge the sets containing \(x\) and \(y\).
MakeSet(x): parent[x] = x; rank[x] = 0

Find(x):  # with path compression
  if parent[x] ≠ x: parent[x] = Find(parent[x])
  return parent[x]

Union(x, y):  # union by rank
  rx = Find(x); ry = Find(y)
  if rx == ry: return
  if rank[rx] < rank[ry]: swap(rx, ry)
  parent[ry] = rx
  if rank[rx] == rank[ry]: rank[rx]++

DAG Shortest Paths

§DAG — Linear-Time Shortest Paths on a DAG

For a DAG with arbitrary (possibly negative) weights, shortest paths from \(s\) can be found in \(O(n+m)\) — faster than Dijkstra or Bellman-Ford.

DAG-ShortestPaths(G, w, s)
1   Topologically sort V into \(v_1, v_2, \ldots, v_n\)
2   for each v: d[v] = ∞; d[s] = 0
3   for each u in topological order:
4       for each v ∈ Adj[u]:
5           if d[u] + w(u,v) < d[v]: d[v] = d[u] + w(u,v); π[v] = u

Why it works: In topological order, when we process \(u\), all predecessors of \(u\) are already finalised. So relaxing \(u\)’s edges gives correct shortest paths. Works with negative edges (no negative cycles possible in a DAG).

\[ T_{\text{DAG-SP}} = O(n + m) \]

All-Pairs Shortest Paths: Floyd-Warshall

§FW1 — Problem & Idea

Problem

Given a weighted digraph \(G=(V,E,w)\) (negative weights allowed, no negative cycles), find the shortest path \(\delta(u,v)\) for every pair \((u,v)\).

Key idea (DP on intermediate vertices): Let \(d^{(k)}[i][j]\) = length of the shortest path from \(i\) to \(j\) using only vertices \(\{1,2,\ldots,k\}\) as intermediates.

\[ d^{(0)}[i][j] = \begin{cases} 0 & i = j \\ w(i,j) & (i,j)\in E \\ \infty & \text{otherwise} \end{cases} \] \[ d^{(k)}[i][j] = \min\!\left(d^{(k-1)}[i][j],\; d^{(k-1)}[i][k] + d^{(k-1)}[k][j]\right) \]

Either the shortest path from \(i\) to \(j\) goes through vertex \(k\) or it doesn’t.

§FW2 — Pseudocode

FloydWarshall(W)   # W = n×n weight matrix
1   d = copy of W (with d[i][i]=0, d[i][j]=∞ if no edge)
2   for k = 1 to n:
3       for i = 1 to n:
4           for j = 1 to n:
5               d[i][j] = min(d[i][j], d[i][k] + d[k][j])
6   return d   # d[i][j] = shortest path i→j

Negative cycle detection: After running, if \(d[i][i] < 0\) for any \(i\), there is a negative-weight cycle reachable involving vertex \(i\).

Path reconstruction: Maintain a predecessor matrix \(\pi[i][j]\): the vertex before \(j\) on the shortest path from \(i\) to \(j\). Update \(\pi[i][j] \leftarrow \pi[k][j]\) whenever \(d[i][j]\) is improved via \(k\).

\[ T_{\text{Floyd-Warshall}} = O(n^3) \qquad \text{Space: } O(n^2) \]

Ford-Fulkerson & Runtime

Algorithm, pseudocode, and all runtime proofs from Lecture 5.

§A1 — Ford-Fulkerson Algorithm

FordFulkerson(G, s, t)
1   for each (u,v) ∈ E: f(u,v) = 0    # zero flow
2   Build residual network G_f
3   while ∃ path P from s to t in G_f:
4       x = min c_f(e) over e ∈ P            # bottleneck
5       Augment f along P by x
6       Update G_f
7   return f

Why greedy alone fails: Pushing forward greedily can get stuck before reaching max flow. Backward edges in \(G_f\) allow “undoing” committed flow to reroute it optimally.

§A3 — Fattest Path Runtime Proof

Proposition 1 (Main Claim)

After one fattest-path augmentation, the remaining max flow value in \(G_f\) satisfies:

\[ v(f')_{\text{new}} \le v(f')\!\left(1-\tfrac{1}{m}\right) \]

Proof: Let \(x=\min_{e\in P}c_f(e)\). Define \(S\) = vertices reachable from \(s\) via residual edges with capacity \(>x\). Any edge from \(S\) to \(V\setminus S\) has capacity \(\le x\), so \(c_f(S,V\setminus S)\le m\cdot x\). By Max-Flow Min-Cut: \(v(f')\le m\cdot x\), i.e. \(x\ge v(f')/m\). Augmenting by \(x\) reduces remaining max flow by \(x\ge v(f')/m\). □

After \(k=m\ln v(f^*)\) iterations the remaining max flow satisfies:

\[ v(f^*)\left(1-\tfrac{1}{m}\right)^{m\ln v(f^*)} < v(f^*)\cdot e^{-\ln v(f^*)}=1 \]

Since residual capacities are integers and the remaining max flow is \(<1\), it must be 0. Total runtime:

\[ T_{\text{fattest}}=O(m(m+n)\log v(f^*)) \]

§A4 — Edmonds-Karp Runtime Proof

Lemma 4 — Distances Never Decrease

Let \(d_i(v)\) = unweighted distance from \(s\) to \(v\) in \(G_f^{(i)}\). Then:

\[ d_{i+1}(v)\ge d_i(v) \quad \forall\,i\ge1,\,v\in V \]

Proof: Induction on \(d_i(v)\). Base \(d=0\): \(v=s\), \(d_{i+1}(s)=0\). Step: let \(u\) be predecessor of \(v\) on shortest path in \(G_f^{(i)}\), so \(d_i(u)=d_i(v)-1\).
Case 1: \((u,v)\in G_f^{(i+1)}\): triangle inequality gives \(d_{i+1}(v)\le d_i(u)+1=d_i(v)\). ✓
Case 2: \((u,v)\notin G_f^{(i+1)}\): \((v,u)\) was on the \(i\)-th augmenting path, so \(d_{i+1}(u)=d_{i+1}(v)+1\), giving \(d_{i+1}(v)\le d_i(v)\). ✓ □

Proposition 2 — Edge Disappearances

Any edge \((u,v)\) can disappear from \(G_f\) at most \(n/2\) times.

Proof: Between two consecutive disappearances of \((u,v)\), \(d(u)\) increases by \(\ge 2\) (via Lemma 4). Since \(0\le d(u)\le n-1\), there are at most \(n/2\) disappearances. □

Total edge disappearances \(\le m\cdot(n/2)\); each iteration costs \(O(m)\) (BFS):

\[ T_{\text{Edmonds-Karp}}=O(m^2 n) \]

§A6 — Cheat Sheet

FLOW NETWORK
  Capacity:     0 ≤ f(u,v) ≤ c(u,v)
  Conservation: ∑ f(x,v) = ∑ f(v,y)  for all internal v
  Flow value:   v(f) = ∑ f(s,x) - ∑ f(y,s)

CUT
  c(S,T) = ∑ c(u,v)  for u∈S, v∈T, (u,v)∈E
  Weak duality: v(f) ≤ c(S,T)

MAX-FLOW MIN-CUT:  max v(f) = min c(S,T)

RESIDUAL NETWORK
  Forward:  c_f(u,v) = c(u,v)-f(u,v)  [if f(u,v) < c(u,v)]
  Backward: c_f(v,u) = f(u,v)          [if f(u,v) > 0]

AUGMENTING PATH (bottleneck x = min c_f(e))
  f'(u,v) = f(u,v)+x (forward)  or  f(u,v)-x (backward)
  v(f') = v(f) + x

RUNTIMES
  Naive:         O(v(f*) * m)
  Fattest path:  O(m(m+n) log v(f*))  [x ≥ v(f')/m]
  Edmonds-Karp:  O(m^2 * n)           [dist never decreases]

Permutation && Combinatorics

Formulas for: permutation with/without repetition, identical items, and combinations.

Permutation with Repetition

All data: \(P(n,n)=n^n\). Order matters, elements can repeat.

Selection \(r\): \(P(n,r)=n^r\)

Permutation without Repetition

All data: \(P(n,n)=n!\). Each element used at most once.

Selection \(r\): \(P(n,r)=\dfrac{n!}{(n-r)!}\)

Permutation with Identical Items

All data: \(P(n,n)=\dfrac{n!}{p_1!\,p_2!\cdots p_k!}\)

Selection \(r\): \(P(n,r)=\dfrac{n!}{(n-r)!\,p_1!\cdots p_k!}\)

Combination

All data: \(C(n,n)=\binom{n}{n}=1\). Order doesn't matter.

Selection \(r\): \(C(n,r)=\binom{n}{r}=\dfrac{n!}{r!\,(n-r)!}\)

Properties

\(P(n,r)=C(n,r)\cdot r!\)   \(C(n,0)=1\)
