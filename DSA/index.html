Core Proof Methods (The "Big Five")

— Direct Proof

  1. Assume \(p\) is true.
  2. Use definitions, axioms, earlier theorems.
  3. Deduce \(q\) step by step.
  4. Conclude "Therefore \(p \to q\)".

Example: If \(n = 2k\), then \(n^2 = 4k^2 = 2(2k^2)\) is even.

— Proof by Contraposition

  1. Prove \(\lnot q \to \lnot p\).
  2. Assume \(\lnot q\), derive \(\lnot p\).
  3. Conclude \(p \to q\) holds.

Example: If \(n = 2k+1\) is odd, then \(n^2 = 2(2k^2+2k)+1\) is odd.

— Proof by Contradiction

  1. Assume \(\lnot p\) or \(p \land \lnot q\).
  2. Derive contradiction \(r \land \lnot r\).
  3. Conclude original statement is true.

Example: \(\sqrt{2}\) irrational — assuming \(\sqrt{2}=a/b\) in lowest terms forces both \(a,b\) even. Contradiction. □

— Proof by Cases

  1. Identify mutually exclusive, exhaustive cases.
  2. Prove \(c_i \to q\) for each.

— Mathematical Induction

Weak: Base \(P(n_0)\) → IH \(P(k)\) → Step \(P(k+1)\).   Strong: Assume \(P(a),\dots,P(k)\); prove \(P(k+1)\).

Examples: \(\sum_{i=1}^n i = \frac{n(n+1)}{2}\)  ·  \(3 \mid (n^3-n)\)  ·  \(2^n > n^2\) for \(n \ge 5\).

How it works

Maintains a sorted prefix \(A[:i]\). Each outer iteration \(i\) takes \(A[i]\) and walks left, shifting elements right until it finds the correct insertion point. After iteration \(i\), \(A[:i+1]\) is sorted.

§4 — Guiding Principles

Worst-Case Analysis

Claims hold for every input of size \(n\). Adversary picks worst input; we still meet \(T(n)\).

Asymptotic Analysis

Focus on growth rate as \(n\to\infty\). Ignore constants and lower-order terms.

§5 — Big-O   (Upper Bound)

Formal Definition
\[ T(n) = O(f(n)) \iff \exists\,c>0,\,n_0\ \text{ s.t. }\ \forall n \ge n_0,\quad 0 \le T(n) \le c\cdot f(n) \]

Example proof: \(5n^2+100n+500 = O(n^2)\). Divide by \(n^2\): \(5+\frac{100}{n}+\frac{500}{n^2}\le c\). Choose \(n_0=10 \Rightarrow c=20\). ✓

Non-example: \(n^3-200n^2 \ne O(n^2)\): would require \(n-200\le c\) for all \(n\), impossible.

§6 — Big-Ω   (Lower Bound)

Formal Definition
\[ T(n) = \Omega(f(n)) \iff \exists\,c>0,\,n_0\ \text{ s.t. }\ \forall n \ge n_0,\quad 0 \le c\cdot f(n) \le T(n) \]

§7 — Big-Θ   (Tight Bound)

Formal Definition
\[ T(n)=\Theta(f(n)) \iff T(n)=O(f(n))\ \text{ and }\ T(n)=\Omega(f(n)) \] \[ \iff \exists\,c_1,c_2>0,\,n_0\ \text{ s.t. }\ \forall n\ge n_0,\quad c_1 f(n)\le T(n)\le c_2 f(n) \]

Limit test (Claim 1): If \(\displaystyle\lim_{n\to\infty}\frac{f(n)}{g(n)}=c>0\), then \(f(n)=\Theta(g(n))\).

§11 — Divide-and-Conquer Paradigm

Definition

Given an input of size \(n\), imagine a magic box that solves the same problem for any input strictly smaller than \(n\). Your job: break the big input into smaller pieces, run them through the box, then stitch the results together. The box is not magic — it is the same algorithm called recursively until a trivial base case is reached.

Every divide-and-conquer algorithm performs three steps in the recursive case:

  1. Divide — split the problem into smaller instances of the same problem.
  2. Conquer — solve each subproblem recursively.
  3. Combine — merge the subproblem solutions into a solution for the original.

Recursion bottoms out at the base case, where the input is small enough to solve directly (e.g. \(n\le1\)).

§12 — MergeSort

Algorithm Idea

Split the array in half, recursively sort both halves, then merge the two sorted halves into one sorted array.

MergeSort Pseudocode

MergeSort(A)
1   n = len(A)
2   if n <= 1:               # base case: already sorted
3       return A
4   L = MergeSort(A[0:n/2])   # sort left half
5   R = MergeSort(A[n/2:n])   # sort right half
6   return Merge(L, R)        # merge the two halves

Python notation: \(A[:n/2] = [A[0],\ldots,A[n/2-1]]\), \(A[n/2:] = [A[n/2],\ldots,A[n-1]]\). Integer division: \(n/2\) means \(\lfloor n/2 \rfloor\).

Merge Pseudocode

Merge(L, R)   # Assume L and R are already sorted
1   S = List of size n
2   i = 0; j = 0
3   for k in [0, ..., n-1]:
4       if L[i] < R[j]:
5           S[k] = L[i]
6           i = i + 1
7       else:
8           S[k] = R[j]
9           j = j + 1
10   return S

# Note: pseudocode above is incomplete — handle end-of-L or end-of-R!

How Merge works: Keep two pointers, one per sorted half. At each step, copy the smaller of the two pointed-to elements into the result, then advance that pointer. Repeat until all elements are copied.

§13 — Correctness of MergeSort

Correctness of Merge (induction on iterations)

Loop Invariant (IH)

After iteration \(t\) of the for loop, \(S[:t-1]\) contains the \(t\) smallest elements of \(L\) and \(R\) in sorted order. Moreover, \(L[i]\) and \(R[j]\) are the smallest elements of their arrays not yet copied to \(S\).

  • Base case (\(t=0\)): \(i=j=0\), \(S\) empty. \(L[0]\) and \(R[0]\) are smallest in their arrays. ✓
  • Inductive step: By IH, after iteration \(t-1\), \(S[:t-2]\) holds the \(t-1\) smallest elements. WLOG \(L[i]\le R[j]\) in iteration \(t\). Then \(L[i]\) is the smallest not yet copied; appending it gives \(S[:t-1]\) with the \(t\) smallest elements in sorted order. ✓
  • Conclusion: When Merge terminates (\(t=n,\,k=n-1\)), \(S[:n-1]\) contains all elements of \(L\) and \(R\) in sorted order. □

Correctness of MergeSort (strong induction on \(n\))

Inductive Hypothesis

Whenever MergeSort returns an array of size \(\le i\), that array is sorted.

  • Base case (\(i=1\)): Arrays of length 0 or 1 are trivially sorted. ✓
  • Inductive step: For input of length \(i>1\), both halves have length \(<i\). By strong IH, lines 4 and 5 correctly produce sorted \(L\) and \(R\). Since Merge is correct (proved above), MergeSort returns the sorted array. ✓
  • Conclusion: By induction, MergeSort correctly sorts any array of length \(n\). □

§14 — Running Time of MergeSort

Work per node

In a single call to MergeSort with input of size \(k = n/2^t\):

  • Checking base case: \(O(1)\)
  • Computing midpoint (divide step): \(O(1)\)
  • Running Merge: \(O(k)\) — the for loop runs exactly \(k\) times, each iteration \(O(1)\)

Total work per node with input size \(k\): \(O(1)+O(1)+O(k) = O(k)\).

Recursion Tree

Level 0         [          n          ]      1 problem of size n,       work = C·n
Level 1         [  n/2  ] [  n/2  ]               2 problems of size n/2,   work = C·n
Level 2         [n/4][n/4][n/4][n/4]      4 problems of size n/4,   work = C·n
                                  ⋮
Level t         2ᵗ problems of size n/2ᵗ,                        work = 2ᵗ · C·(n/2ᵗ) = C·n
                                  ⋮
Level log n     n problems of size 1,                             work = C·n

Key insight: Work at every level \(t\) equals \(2^t \cdot C\cdot(n/2^t) = C\cdot n\) — constant per level, independent of \(t\). The two exponentials cancel exactly.

Number of levels: \(\log n + 1\) (tree depth is \(\log n\) since size halves each level).

Claim 1 — MergeSort Runtime (Recursion Tree Proof)

Theorem
\[ T_{\text{MergeSort}}(n) = O(n \log n) \]

Proof:

\[ \text{Total} = (\text{ops per level}) \times (\text{number of levels}) \le C\cdot n \cdot (\log n + 1) = C\cdot(n\log n + n) = O(n\log n) \quad \square \]

Recurrence Relation

Recurrence
\[ T(n) \le 2\,T\!\left(\frac{n}{2}\right) + C\cdot n \qquad T(1) \le C \]

where \(C\) is a constant. For \(n=2^k\): \(T(2^k) \le 2\,T(2^{k-1}) + C\cdot 2^k\).

"Plug-and-Pattern-Match" Proof

Unrolling the recurrence \(k\) times (let \(n=2^k\)):

\[ T(2^k) \le 2^i\,T(2^{k-i}) + i\cdot C\cdot 2^k \]

Setting \(i=k\) and using \(T(2^0)=T(1)\le C\):

\[ T(2^k) \le 2^k\cdot C + k\cdot C\cdot 2^k = C(k+1)2^k = O(2^k\cdot k) \]

Since \(n=2^k\) and \(k=\log n\):

\[ T(n) = O(n\log n) \quad \square \]

"Induction-from-the-Ansatz" Proof

Guess \(T(2^k)\le C(k+1)2^k\) and prove by induction.

  • Base (\(k=0\)): \(T(1)\le C = C\cdot1\cdot1 = C(0+1)2^0\). ✓
  • IH: \(T(2^{k-1})\le C\cdot k\cdot 2^{k-1}\).
  • Step: \[ T(2^k)\le 2T(2^{k-1})+C\cdot 2^k \le 2\cdot Ck\cdot 2^{k-1}+C\cdot 2^k = Ck\cdot 2^k + C\cdot 2^k = C(k+1)2^k \quad\square \]

§16 — Practical Notes on MergeSort

  • External sorting: Merge only needs the "tips" (front elements) of each half — great when data doesn't fit in memory.
  • Predictable runtime: \(\Theta(n\log n)\) in all cases — no worst-case blowup unlike InsertionSort.
  • Parallelism: Recursive calls can run in parallel. Naive parallelisation gives \(\Theta(n)\) (sequential merge bottleneck). Parallel merge reduces this to \(\Theta(n/\log^2 n)\) with infinite CPUs.
  • Design question to ask: If I could solve this problem for any portion of the input independently, can I stitch those solutions back together?

Max Flow & Min Cut — Graph Definitions

Graph-theoretic definitions from Lecture 5: flow networks, cuts, residual networks, and the Max-Flow Min-Cut theorem.

§G1 — Flow Network

Definition

A flow network is a directed graph \(G=(V,E)\) with a capacity function \(c:E\to\mathbb{R}_{\ge0}\), a single source \(s\in V\) (no incoming edges), and a single sink \(t\in V\) (no outgoing edges). All other vertices are internal.

  • No edge enters \(s\); no edge leaves \(t\).
  • Every vertex has at least one incident edge.
  • All capacities are integers (in the standard formulation).

§G2 — Flow & Its Value

s-t Flow

A function \(f:E\to\mathbb{R}_{\ge0}\) satisfying:

Capacity Constraint

\[ \forall\,(u,v)\in E \qquad 0 \le f(u,v) \le c(u,v) \]

Flow Conservation

\[ \forall\,v\in V{-}\{s,t\} \quad \sum_{x\in N_{\mathrm{in}}(v)}f(x,v)=\sum_{y\in N_{\mathrm{out}}(v)}f(v,y) \]

Value of a Flow

\[ v(f)=\sum_{x\in N_{\mathrm{out}}(s)}f(s,x)-\sum_{y\in N_{\mathrm{in}}(s)}f(y,s) \]

Max Flow Problem: Find the \(s\)-\(t\) flow \(f\) that maximises \(v(f)\).

§G3 — s-t Cut

Definition

An \(s\)-\(t\) cut is a partition \(V=S\cup T\), \(S\cap T=\emptyset\), \(s\in S\), \(t\in T\). Its capacity counts only edges from \(S\) to \(T\):

\[ c(S,T)=\sum_{\substack{(u,v)\in E\\u\in S,\,v\in T}}c(u,v) \]

Min Cut Problem: Find the \(s\)-\(t\) cut of minimum capacity (cheapest disconnection of \(s\) from \(t\)).

Multiple sources/sinks: Add a super-source with \(\infty\)-capacity edges to all real sources; similarly for a super-sink.

§G4 — Max-Flow Min-Cut Theorem

Lemma 1 — Weak Duality

For any flow \(f\) and any cut \((S,T)\):

\[ v(f)\le c(S,T) \]
Corollary 1

If \(v(f)=c(S,T)\) for some \(f\) and \((S,T)\), then \(f\) is a maximum flow and \((S,T)\) is a minimum cut.

Theorem 1 — Max-Flow Min-Cut
\[ \max_f\,v(f)=\min_{(S,T)}c(S,T) \]

In every flow network, max \(s\)-\(t\) flow value = min \(s\)-\(t\) cut capacity.

§G5 — Residual Network

Definition: \(G_f\)

Given \(G\) and flow \(f\), the residual network \(G_f\) has the same vertices. For each edge \((u,v)\in E\):

ConditionEdge in \(G_f\)TypeResidual capacity
\(f(u,v)<c(u,v)\)\((u,v)\)Forward\(c_f(u,v)=c(u,v)-f(u,v)\)
\(f(u,v)>0\)\((v,u)\)Backward\(c_f(v,u)=f(u,v)\)

\(G_f\) contains only edges with strictly positive residual capacity. Each original edge yields at most 2 edges in \(G_f\), so \(|E_{G_f}|\le 2|E|\).

§G6 — Augmenting Path

Definition

An augmenting path is any path from \(s\) to \(t\) in \(G_f\). Its bottleneck is \(x=\min_{e\in P}c_f(e)\).

\[ f'(u,v)=\begin{cases}f(u,v)+x&(u,v)\in P\text{ (forward)}\\f(u,v)-x&(v,u)\in P\text{ (backward)}\\f(u,v)&\text{otherwise}\end{cases} \] \[ v(f')=v(f)+x>v(f) \]
Lemma 2

If \(t\) is not reachable from \(s\) in \(G_f\), then \(f\) is a max flow. (Let \(S=\) vertices reachable from \(s\); then \(v(f)=c(S,T)\).)

Lemma 3

If \(G_f\) has any path from \(s\) to \(t\), we can augment \(f\) to \(f'\) with \(v(f')>v(f)\).

Graph Theory — Foundations

All definitions, types, walks, and connectivity from Lecture 4 (Slides).

§L4-1 — What Is a Graph?

Definition

A graph \(G=(V,E)\) is a set of vertices (nodes) \(V\) and a set of edges \(E\), where each edge connects a pair of vertices. It models pairwise relationships between items.

  • Order of \(G\): the number of vertices \(|V|\).
  • Size of \(G\): the number of edges \(|E|\).
  • A graph of order 0 or 1 is called trivial.
  • Drawing: vertices as circles, edges as lines — only connectivity matters, not geometry.

Origin: Leonhard Euler (1736) solved the Königsberg Seven Bridges Problem by abstracting land masses as vertices and bridges as edges — the first theorem in graph theory.

Theorem

The sum of all vertex degrees equals twice the number of edges. Consequence: the number of odd-degree vertices is always even.

Ford-Fulkerson & Runtime

Algorithm, pseudocode, and all runtime proofs from Lecture 5.

§A1 — Ford-Fulkerson Algorithm

FordFulkerson(G, s, t)
1   for each (u,v) ∈ E: f(u,v) = 0    # zero flow
2   Build residual network G_f
3   while ∃ path P from s to t in G_f:
4       x = min c_f(e) over e ∈ P            # bottleneck
5       Augment f along P by x
6       Update G_f
7   return f

Why greedy alone fails: Pushing forward greedily can get stuck before reaching max flow. Backward edges in \(G_f\) allow “undoing” committed flow to reroute it optimally.

§A3 — Fattest Path Runtime Proof

Proposition 1 (Main Claim)

After one fattest-path augmentation, the remaining max flow value in \(G_f\) satisfies:

\[ v(f')_{\text{new}} \le v(f')\!\left(1-\tfrac{1}{m}\right) \]

Proof: Let \(x=\min_{e\in P}c_f(e)\). Define \(S\) = vertices reachable from \(s\) via residual edges with capacity \(>x\). Any edge from \(S\) to \(V\setminus S\) has capacity \(\le x\), so \(c_f(S,V\setminus S)\le m\cdot x\). By Max-Flow Min-Cut: \(v(f')\le m\cdot x\), i.e. \(x\ge v(f')/m\). Augmenting by \(x\) reduces remaining max flow by \(x\ge v(f')/m\). □

After \(k=m\ln v(f^*)\) iterations the remaining max flow satisfies:

\[ v(f^*)\left(1-\tfrac{1}{m}\right)^{m\ln v(f^*)} < v(f^*)\cdot e^{-\ln v(f^*)}=1 \]

Since residual capacities are integers and the remaining max flow is \(<1\), it must be 0. Total runtime:

\[ T_{\text{fattest}}=O(m(m+n)\log v(f^*)) \]

§A4 — Edmonds-Karp Runtime Proof

Lemma 4 — Distances Never Decrease

Let \(d_i(v)\) = unweighted distance from \(s\) to \(v\) in \(G_f^{(i)}\). Then:

\[ d_{i+1}(v)\ge d_i(v) \quad \forall\,i\ge1,\,v\in V \]

Proof: Induction on \(d_i(v)\). Base \(d=0\): \(v=s\), \(d_{i+1}(s)=0\). Step: let \(u\) be predecessor of \(v\) on shortest path in \(G_f^{(i)}\), so \(d_i(u)=d_i(v)-1\).
Case 1: \((u,v)\in G_f^{(i+1)}\): triangle inequality gives \(d_{i+1}(v)\le d_i(u)+1=d_i(v)\). ✓
Case 2: \((u,v)\notin G_f^{(i+1)}\): \((v,u)\) was on the \(i\)-th augmenting path, so \(d_{i+1}(u)=d_{i+1}(v)+1\), giving \(d_{i+1}(v)\le d_i(v)\). ✓ □

Proposition 2 — Edge Disappearances

Any edge \((u,v)\) can disappear from \(G_f\) at most \(n/2\) times.

Proof: Between two consecutive disappearances of \((u,v)\), \(d(u)\) increases by \(\ge 2\) (via Lemma 4). Since \(0\le d(u)\le n-1\), there are at most \(n/2\) disappearances. □

Total edge disappearances \(\le m\cdot(n/2)\); each iteration costs \(O(m)\) (BFS):

\[ T_{\text{Edmonds-Karp}}=O(m^2 n) \]

§A6 — Cheat Sheet

FLOW NETWORK
  Capacity:     0 ≤ f(u,v) ≤ c(u,v)
  Conservation: ∑ f(x,v) = ∑ f(v,y)  for all internal v
  Flow value:   v(f) = ∑ f(s,x) - ∑ f(y,s)

CUT
  c(S,T) = ∑ c(u,v)  for u∈S, v∈T, (u,v)∈E
  Weak duality: v(f) ≤ c(S,T)

MAX-FLOW MIN-CUT:  max v(f) = min c(S,T)

RESIDUAL NETWORK
  Forward:  c_f(u,v) = c(u,v)-f(u,v)  [if f(u,v) < c(u,v)]
  Backward: c_f(v,u) = f(u,v)          [if f(u,v) > 0]

AUGMENTING PATH (bottleneck x = min c_f(e))
  f'(u,v) = f(u,v)+x (forward)  or  f(u,v)-x (backward)
  v(f') = v(f) + x

RUNTIMES
  Naive:         O(v(f*) * m)
  Fattest path:  O(m(m+n) log v(f*))  [x ≥ v(f')/m]
  Edmonds-Karp:  O(m^2 * n)           [dist never decreases]

Graph Representations

Data structures for storing graphs and their trade-offs, from Lecture 4.

§R1 — Adjacency Matrix

Definition

An \(n\times n\) matrix \(A\) where:

\[ A[i][j] = \begin{cases}1 & \text{if }(i,j)\in E \\ 0 & \text{otherwise}\end{cases} \]

For a multigraph: \(A[i][j]\) = number of edges between \(i\) and \(j\).
For a digraph: \(A[i][j]=1\) means arc from \(i\) (source/row) to \(j\) (destination/column).

  • Symmetric for undirected graphs: \(A[i][j]=A[j][i]\).
  • Space: \(\Theta(n^2)\).
  • Edge query \((u,v)\in E\): \(O(1)\).
  • Neighbour query (list all neighbours of \(v\)): \(O(n)\).
  • Slow to add/remove vertices — matrix must be resized/copied.

§R2 — Adjacency List

Definition

An array of \(n\) lists. \(\text{Adj}[v]\) contains all vertices adjacent to \(v\).

For a digraph: \(\text{Adj}[v]\) stores only the out-neighbours of \(v\).

  • Space: \(\Theta(n+m)\) where \(m=|E|\) — efficient for sparse graphs.
  • Neighbour query: \(O(\deg(v))\).
  • Edge query \((u,v)\in E\): \(O(\deg(u))\) — must scan the list.
  • Slow to remove a vertex — must find and delete it from all neighbour lists.
  • Preferred for sparse graphs and most algorithm implementations.

Dense vs Sparse: A dense graph has \(m = \Theta(n^2)\) edges; a sparse graph has \(m = O(n)\) or \(O(n\log n)\). For real-world networks (web, social, biological) the graph is almost always sparse — adjacency list wins.

Note from lecture: Adjacency list is assumed for the rest of the course.

Permutation && Combinatorics

Formulas for: permutation with/without repetition, identical items, and combinations.

Permutation with Repetition

All data: \(P(n,n)=n^n\). Order matters, elements can repeat.

Selection \(r\): \(P(n,r)=n^r\)

Permutation without Repetition

All data: \(P(n,n)=n!\). Each element used at most once.

Selection \(r\): \(P(n,r)=\dfrac{n!}{(n-r)!}\)

Permutation with Identical Items

All data: \(P(n,n)=\dfrac{n!}{p_1!\,p_2!\cdots p_k!}\)

Selection \(r\): \(P(n,r)=\dfrac{n!}{(n-r)!\,p_1!\cdots p_k!}\)

Combination

All data: \(C(n,n)=\binom{n}{n}=1\). Order doesn't matter.

Selection \(r\): \(C(n,r)=\binom{n}{r}=\dfrac{n!}{r!\,(n-r)!}\)

Properties

\(P(n,r)=C(n,r)\cdot r!\)   \(C(n,0)=1\)
